Quartoで構築したこと

Quartoとは

Quarto（クワルト）は、Rや Python のコードを含むドキュメントをWebサイト、PDF、プレゼンテーションなど様々な形式に変換できるオープンソースのパブリッシングシステムです。このサイトはRStudio上でQuartoを使って構築されています。

仕組み説明
.qmdファイルMarkdown + Rコードを書いたテキストファイル。これがページの素材
Rコードチャンク```{r}で囲まれたRコードが自動実行され、表・グラフが生成される
_quarto.ymlサイト全体の設定ファイル（タイトル・ナビ・フッター・テーマなど）
quarto render全ページをHTMLに変換するコマンド。出力先は _site/フォルダ

必要なソフトウェア

  1. R（4.0以上）— https://cran.r-project.org/ からインストール
  2. RStudio（2026.05系以上推奨）— https://posit.co/download/rstudio-desktop/
  3. Git（GitHub連携のため）— https://git-scm.com/
  4. 必要なRパッケージ（初回のみコンソールで実行）
R コンソール
install.packages(c("tidyverse", "knitr", "exiftoolr", "magick"))
⚠️ ExifToolRの追加設定 exiftoolrパッケージはギャラリー写真の振り分けに使用します。初回のみ以下も実行してください：
exiftoolr::install_exiftool()

GitHub Pagesへの公開

仕組みの概要

このサイトは GitHub Pages を使って無料で公開されています。GitHubのリポジトリ（DaisukeOSUアカウント）に _site/ フォルダの内容をアップロードすることで、ウェブに公開されます。

📌 前提条件 Gitのインストールと、GitHubアカウント（DaisukeOSU）への認証設定が必要です。初回設定については赤石先生に確認してください。

公開の基本的な流れ

  1. RStudioで編集（CSVやqmdを更新）
  2. RStudioのTerminalで quarto render を実行 → _site/ が更新される
  3. _site/ フォルダの内容をGitHubにpush
  4. 数分後にサイトが更新される

Terminal での操作手順

RStudioの下部にある「Terminal」タブを開いて、以下を順番に実行します。

Terminal（毎回の更新作業）
# 1. 変更内容を確認（どのファイルが変わったか確認できる）
git status

# 2. 全ての変更をステージング
git add .

# 3. コミット（"" の中は変更の説明を日本語で書いてOK）
git commit -m "ニュースを更新"

# 4. GitHubにアップロード（push）
git push origin main
✅ ショートカット：毎回の更新はこの4行だけ覚えればOK git add . → git commit -m "説明" → git push origin main

GitHub Pagesの設定確認

GitHubリポジトリの Settings → Pages で、以下が設定されていることを確認してください。

設定項目値
Source（ソース）Deploy from a branch
Branchmain ／ /docs または /(root)
⚠️ _quarto.yml の output-dir に注意 現在 output-dir: _site となっています。GitHub Pages の設定が /docs になっている場合は output-dir: docs に変更が必要です。赤石先生に現在の設定を確認してください。

ビルド手順（render）

方法①：RStudioのボタンから（初心者向け）

  1. RStudioで files.Rproj を開いてプロジェクトを起動
  2. 右上の「Build」タブをクリック
  3. 「Render Website」ボタンをクリック
  4. Viewer ペインにサイトのプレビューが表示される

方法②：Terminalから（推奨）

Terminal
# サイト全体をビルド
quarto render

# 特定のページだけビルド（高速）
quarto render news.qmd

# プレビュー（変更を自動検知して更新）
quarto preview
💡 quarto preview が便利 quarto preview を実行するとブラウザが自動で開き、ファイルを保存するたびにリアルタイムでプレビューが更新されます。編集作業中は常にこれを使うと便利です。

ビルド時のエラー例

ニュースの更新

ニュースは news.csv を編集するだけで更新できます。RStudioやExcel・テキストエディタで開いて編集できます。

news.csv のフォーマット

news.csv の構造
date,type,content,url
2026/05/31,研究,ウェブサイトを立ち上げました。,
2026/05/30,メディア,読売新聞の取材にコメントしました。,https://www.osaka-sandai.ac.jp/...
列名内容使用できる値
date日付2026/06/01 の形式
type種別バッジ論文・受賞・研究・募集・メディア
contentニュース本文自由記述。HTMLタグも使用可
urlリンク先URL（任意）URLがあればcontentがリンクになる。なければ空欄
✅ バッジの色について typeに入れた値によってバッジの色が自動で変わります：
🟢 論文（緑）　🟡 受賞（黄）　🔵 研究（青）　🔴 募集（赤）　🟣 メディア（紫）

注意事項

⚠️ カンマや引用符を含む場合 contentにカンマ（,）や引用符を含む場合は、セル全体をダブルクォートで囲んでください：
"日本生命財団、環境問題研究助成に採択されました。"
内側に引用符が必要な場合は ""（二重）にします。

CSVを保存したら、quarto render で再ビルドして公開します。

論文一覧の更新手順

論文一覧は researchmap からエクスポートしたCSVファイルを使っています。

  1. researchmap にログインし、「論文」ページを開く
  2. 画面右上の「エクスポート」→「CSV形式」を選択してダウンロード

  4. publications.qmd 内の読み込みファイル名を確認・更新
publications.qmd 内の該当箇所
papers <- read_csv(
  "rm_published_papers20260531.csv",  # ← ここのファイル名を更新する
  skip = 1,
  col_types = cols(.default = col_character())

💡 ファイル名の日付を更新する エクスポートするたびにファイル名の日付部分が変わります（例：rm_published_papers20260601.csv）。コード内のファイル名もあわせて変更してください。

ギャラリーの更新

ギャラリーは写真を追加するだけで自動的に年月別に整理される仕組みになっています。

写真追加の手順

  1. 追加したい写真（JPG/PNG）を gallery/ フォルダにコピーする
  2. RStudioで gallary.R を開いて全体を実行（Ctrl+A → Ctrl+Enter）
  3. スクリプトが自動で写真を圧縮し、撮影日時に基づいて photos/年/月/ に振り分ける
  4. quarto render でビルドして確認・公開

gallary.R の動作説明

処理内容説明
重複チェックすでに photos/ にあるファイルは処理をスキップ（二重追加の防止）
圧縮長辺1200pxにリサイズし、品質80%で再保存（容量削減）
日付取得写真のExifデータから撮影日時を自動取得
フォルダ振り分けphotos/2025/10/ のように年月フォルダに自動コピー
⚠️ 日付不明の写真はスキップされる スマホで撮影した写真には通常Exifデータが入っています。スクリーンショットなど日付情報がない場合はスキップされます。その場合は手動で photos/年/月/ フォルダに直接コピーしてください。

フォントの変更方法

現在は「Zen Maru Gothic」（丸みのある日本語フォント）を使用しています。変更する場合は以下を編集します。

styles.css — フォント設定
/* ここのURLを変えると別のGoogleフォントを使える */
@import url('https://fonts.googleapis.com/css2?family=Zen+Maru+Gothic:wght@400;500;700&display=swap');

body {
  font-family: 'Zen Maru Gothic', serif;  /* ← フォント名をここに */
}
💡 Google Fontsで日本語フォントを探す https://fonts.google.com/?subset=japanese でフィルター「Japanese」を選ぶと日本語対応フォントが一覧できます。

トラブルシューティング

よくあるトラブルと対処法

① quarto render がエラーで止まる

まずエラーメッセージをよく読み、どのファイルの何行目でエラーが出たかを確認します。多くの場合、RパッケージのインストールまたはCSVの書式ミスが原因です。

② 写真がギャラリーに表示されない

確認事項対処
ファイルが photos/年/月/ にあるか手動でフォルダを確認
ファイル名に日本語や特殊文字がないか英数字・アンダースコアのみに変更
Exifデータが正しいかgallary.R 実行時のメッセージを確認

③ GitHubにpushできない

Terminal
# まず状況確認
git status

# 認証エラーの場合、GitHubの認証情報を再設定
git config --global user.email "your-email@example.com"
git config --global user.name "Your Name"

# それでも解決しない場合
# GitHubの「Settings → Developer settings → Personal access tokens」でトークンを発行

④ サイトの一部が更新されない

ブラウザのキャッシュが原因のことがあります。Ctrl+Shift+R（強制リロード）で解決することが多いです。

⑤ ニュースのバッジ色が「灰色」になる

news.csv の type 列の値が 論文・受賞・研究・募集・メディア 以外になっているときです。スペルを確認してください。
